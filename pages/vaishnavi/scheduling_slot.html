Slot Scheduling System

Business Case Relating to SDGs

Target 12.3: Halve per capita global food waste by reducing losses in production and supply chains

Efficient logistics scheduling ensures that perishable goods are transported and processed quickly, minimizing waste during transit within SEZs.

Target 11.2: Provide access to safe, affordable, and sustainable transport systems for all

Optimized traffic flow and reduced congestion improve goods movement efficiency, enhancing the accessibility of transport systems within SEZs.

To make the scheduling system more efficient, we can apply algorithms specifically designed for job scheduling, resource allocation, or load balancing.

1. Greedy Algorithm for Earliest Available Slot

This algorithm uses a priority queue (min-heap) to store docks based on their available time. It efficiently fetches the dock with the earliest availability in O(log D), where D is the number of docks.

Objective: Schedule trucks as they arrive using available docks.

2. Interval Partitioning

This algorithm assigns trucks to docks to avoid overloading multiple trucks at the same dock simultaneously. It minimizes overlap while using the fewest docks possible.

Objective: Minimize overlap and use as few docks as possible.

Output:

Truck 1 scheduled from 09:00 to 13:00 at Dock 0
Truck 2 scheduled from 09:30 to 10:30 at Dock 1
Truck 3 scheduled from 10:00 to 12:00 at Dock 2
Truck 4 scheduled from 10:30 to 11:30 at Dock 1
Truck 5 scheduled from 13:00 to 16:00 at Dock 0

Code: Interval Partitioning Code

3. Load Balancing

This algorithm distributes workload evenly across docks, ensuring efficient utilization of resources.

Objective: Distribute workload evenly across docks.

Efficiency Analysis

The following outlines the time and space efficiency of the key algorithms used in the scheduling system:

  • Greedy Algorithm: O(log D) time complexity due to priority queue operations
  • Interval Partitioning: O(T log D) time complexity for sorting trucks and assigning docks
  • Load Balancing: O(T log D) for balancing the load across docks
